模块化

发表于 2017-07-20 | 分类于 规范

js不断被类聚和抽象来，从另一个角度来看，它也道出了js先天就缺乏的一项功能:模块

相比于其他高级语言，java有类文件，python有import，ruby有require，php有include和require而js通过<script>标签引入代码的方式显得杂乱无章，毫无组织和约束能力。因此提出了CommonJS规范

知识点

  • 不使用模块化的情况
  • 使用模块化
  • AMD
  • CommonJS

  • 不使用模块化的情况

  • 使用模块化

  • AMD (异步模块定义)

    • require.js 一个规范
    • 全局 define 函数
    • 全局 require 函数
    • 依赖js会自动、异步加载
  • CommonJS
    • nodejs模块化规范，现被大量用前段，原因：
      • 前段开发依赖的插件和库，都可以从npm获取
      • 构建工具的高度自动话，使得使用npm的成本非常低
      • CommonJS不会异步加载JS,和AMD不同，而是同步一次性加载出来

浏览器存储

发表于 2017-07-09 | 分类于 本地存储

知识点

  • cookie
  • sessionStorage和localStorage

  • 使用document.cookie = … 获取和修改即可

  • cookie

    • 本身用于客户端和服务器端通信
    • 但是它又本地存储的功能，于是就被借用
  • cookie用于存储的缺点
    • 存储量小，只有4KB
    • 所有http请求都带着，会影响获取资源的效率
    • API简单，需要封装才能用 document.cookie = …
    • 后来h5出了sessionStorage和localStorage可以解决
  • sessionStorage和localStorage
    • HTML5专门为存储儿设计，最大容量5M
    • API简答易用
    • localStorage.setItem(key, value);localStorage.getItem(key)
  • sessionStorage和localStorage区别
