密码安全

发表于 2017-05-19 | 分类于 web安全
密码泄露渠道
  • 数据库被偷
  • 服务器被入侵
  • 通信过程中被窃听
密码安全防御：
  • 不要明文存醋密码
哈希算法：
  • 明文、密文一一对应
  • 雪崩效应 不同密码(即明文)对应的密文可能相同，但是几率很小
  • 密文无法反推出明文
  • 密文的长度是固定的
  • 常见的哈希算法md5、sha1、sha256

但是，若有一张表收集了所有明文以及对应的密文，那么只要将用户密码的密文去查这张表，就可以得到相应的明文，这种表叫彩虹表，所以哈希算法不绝对安全的，建议用户的明文密码要足够复杂，因为彩虹表的样本不会太大，简单的密码才会被查出来，也可以通过组合md5、sha1、sha256这三种的方式存储用于密码，还可以通过加盐帮用户提高密码的复杂度，这个盐是随机数可以是时间戳。
