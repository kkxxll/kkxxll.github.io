es6正则扩展

发表于 2017-08-04 | 分类于 es6 
{
  //es5
  let regex1 = new RegExp('abc', 'i'); //第一个参数是字符串，第二个是修饰符
  let regex2 = new RegExp(/abc/i);  //第一个参数是正则表达式，不接受第二个参数，否则会报错
  console.log(regex1.test('abc123'),regex2.test('abc123'));     //true true

  //es6
  let regex3 = new RegExp(/abc/ig, 'i');//原有正则对象的修饰符是ig，它会被第二个参数i覆盖

  console.log(regex3.test('abc123'));                           //true
  console.log(regex3.flags)                                     //i
}
不同的修饰符g与y

相同：g，y都是全局匹配
不同：g是从任何位置开始匹配，而y是从紧跟着的第一个位置匹配

可以用sticky查看是否用y模式即是否开启了粘连模式

console.log(r1.sticky, r2.sticky); //false true

u修饰符(用于unico编码)

{
  console.log(/^\uD83D/.test('\uD83D\uDC2A'));   //true
  console.log(/^\uD83D/u.test('\uD83D\uDC2A'));  //false
}

u修饰符把’\uD83D\uDC2A’当成一整个字符，所以第二个匹配不到

{
  console.log(/\u{61}/.test('a'));      //false
  console.log(/\u{61}/u.test('a'));     //true
}

大括号表示Unicode字符，加上以后u才能识别
61对应的unicode是a,u修饰符会识别{61}，所以第二个可以匹配

console.log(/\u{20BB7}/u.test('𠮷')); // true
// 点（.）字符不能识别码点大于0xFFFF的Unicode字符，必须加上u修饰符。
let s = '𠮷';
console.log('大于0xFFFF的Unicode字符',/^.$/.test(s)); // false
console.log('使用u字符',/^.$/u.test(s)); // true

// 使用u修饰符后，所有量词都会正确识别大于码点大于0xFFFF的Unicode字符。
console.log('量词',/a{2}/.test('aa')); // true
console.log('量词',/a{2}/u.test('aa')); // true
console.log('量词',/𠮷{2}/.test('𠮷𠮷')); // false
console.log('量词',/𠮷{2}/u.test('𠮷𠮷')); // true

正则表达式中，点（.）是一个特殊字符，代表任意的单个字符，但是行终止符（line terminator character）除外
以下都是行终止符
U+000A 换行符（\n）
U+000D 回车符（\r）
U+2028 行分隔符（line separator）
U+2029 段分隔符（paragraph separator）
