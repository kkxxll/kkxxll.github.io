基于MV*模式的Vue框架

发表于 2017-07-19 | 分类于 vue

MV*框架特点:

  • Model绑定View
  • 没有控制器概念
  • 数据驱动（不需要js来操作dom），状态管理来控制View
  • 组件化。即做一个登陆模块可以用在很多页面里］

vue是通过MVVM的数据绑定实现自动同步

view(DOM)—ViewModel(Vue)—Model(POJO即原生js对象)

ViewModel监听View后去更新Model

vue实现双向数据绑定的原理
利用虚拟DOM实现快速渲染

用JS对象表示DOM结构，那么当数据状态发生变化而需要改变DOM结构时，我们先通过JS对象表示的虚拟DOM计算出实际DOM需要做的最小变动，然后再操作实际DOM，从而避免了粗放式的DOM操作带来的性能问题。

前端路由指的是在单页面应用中，大部分页面结构不变，只改变部分内容时使用

前端路由与路由的区别

路由是根据不同的url地址展示不同的内容或页面

前端路由是把不同路由对应的内容或页面交给前端来做，之前是通过服务器根据url的不同返回不同的页面实现的

前端路由优点：

用户体验好，不需要每次都从服务器全部获取，快速展现给用户

缺点：

不利于SEO(可能只有首页会被搜索引擎收录，后面的都是模拟出来的不会被收录)
使用浏览器的前进后退键，因为不是一个真实的页面，所以没有缓存，会去重新发送请求
单页面无法记住之前滚动的位置

动态路由

/user/:username {username: ‘kk’} /user/kk
/user/:username/post/:post_id {username: ‘kk’, post_id:1} /user/kk/post/1

异步和单线程
